# Our Story Page Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a fully customizer-editable "Our Story" Shopify page at `/pages/our-story` with 6 sections: Hero, Origin Story, Value Pillars, Founder Quote, Gallery Strip, and CTA.

**Architecture:** Two new files — `sections/our-story.liquid` (entirely self-contained: Google Fonts, `:root` design tokens, all CSS, all Liquid HTML, full schema) and `templates/page.our-story.json` (routes the URL to the section). The template is created **last**, after the schema is complete, to avoid Shopify rejecting template block references that aren't yet declared in the schema.

**Tech Stack:** Shopify Liquid, inline CSS (no external stylesheet), Shopify theme customizer schema (JSON). No JS required. Verify with `shopify theme dev --store supply-district-8763.myshopify.com`.

**Spec:** `docs/superpowers/specs/2026-03-13-our-story-page-design.md`

---

> **⚠️ Human prerequisite (do once before verifying in browser):**
> In Shopify admin → Pages → create a new page titled "Our Story" with handle `our-story`. Assign it the template `page.our-story`. You cannot do this via the CLI — it must be done in the Shopify admin UI. The page only needs to exist; leave the body blank.

---

> **Note on testing:** Shopify Liquid has no unit test framework. Each task's verification step is a browser check against the dev server. Run `shopify theme dev --store supply-district-8763.myshopify.com` once before starting and keep it running throughout.

---

> **Note on the section wrapper:** The schema declares `"tag": "div"` and `"class": "cvh-os-wrap"`. Shopify automatically wraps the section's rendered HTML in `<div class="cvh-os-wrap">`. Do **not** add an explicit `<div class="cvh-os-wrap">` in the Liquid HTML — that would produce a double-wrapped element.
